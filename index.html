HTML

  • The head element contains information about the webpage, including the title and metadata
  • The body element represents the visible content shown to the user.
  • HTML is the framework of a website.
  • !DOCTYPE tells the browser what code it should expect to recieve. Doctype is not case sensitive.
  • Tags are what is within the '< >' signs, elements are the whole thing.
  • Semantic elements are tags that have no effect on the webpage, they are just used for organization. Like main, header, body and section

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The terminology and formatting for css rule goes as such: selector{property: value} and the whole thing is referred to as a "declaration"
  • When entering property values that style the top, bottom, left, and right; two values can be set, the first defining top and bottom, the other defining left and right.
  • Setting "auto" to sizing declarations will auto set the value of the size to what fits the webpage or div.

Git and CLI

Git

  • git status: checks what branch we are currently working on
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git cd: changes the directory the computer works under. "git cd .." will go backwards.
  • git add -A: add command will add modifications in the current working branch to the staging area. The -A glag indicates that we want to add all changes. (The staging area informs git that the changes will be added in the next commit.)
  • git commit -m "title": commits currently staged changes with a title that describes the changes
  • git pull origin main: good practice to double check if local branch is in sync with the remote base branch in GitHub. pull recieves a branch's modifications into the local environment. origin indicates the source of the pull will be in the GitHub repo. main indicates the branch.
  • git push origin branch: push will push the local changes to the remote GitHub branch.

CLI

  • code . : opens up code editor
  • pwd: displays the current working directory
  • ls: displays list of files and folders in the current directory
  • mkdir: creates directory
  • touch: creates file inside directory
